## 1. Подготовка рабочего места с использованием ОС Windows 10

Приводимые ниже шаги позволят установить и настроить все необходимые
компоненты для прохождения курса.

### 1.1 Установка VirtualBox

Первым шагом является установка средства виртуализации **VirtualBox**.
Данное средство требуется для создания и запуска виртуальной машины,
которая содержит гостевую ОС и набор средств разработки, что позволяет
производить компиляцию исходного кода, запуск исполняемых файлов и
загрузку/выгрузку модулей ядра ОС изолированно от хостовой ОС.

#### 1.1.1 Получение установочного файла VirtualBox

Установочный файл **VirtualBox** можно получить с официального сайта
[VirtualBox][Section_1.1.1_Link_1].

![Раздел 1.1.1 Рисунок 1][Section_1.1.1_Figure_1]

![Раздел 1.1.1 Рисунок 2][Section_1.1.1_Figure_2]

Находясь на официальном сайте **VirtualBox**:

1. Кликнете по ссылке **Download VirtualBox 6.1**.
2. Кликнете по ссылке **Windows hosts** в разделе
   **VirtualBox 6.1.2 platform packages**.

Примечание: Актуальная версия может отличаться от представленной в примере.

### 1.2 Настройка VirtualBox

#### 1.2.1 Создание виртуальных сетевых интерфейсов

![Раздел 1.2.1 Рисунок 1][Section_1.2.1_Figure_1]

![Раздел 1.2.1 Рисунок 2][Section_1.2.1_Figure_2]

![Раздел 1.2.1 Рисунок 3][Section_1.2.1_Figure_3]

![Раздел 1.2.1 Рисунок 4][Section_1.2.1_Figure_4]

![Раздел 1.2.1 Рисунок 5][Section_1.2.1_Figure_5]

![Раздел 1.2.1 Рисунок 6][Section_1.2.1_Figure_6]

Для создания виртуальных сетевых интерфейсов выполните следующие действия:

1. Выберите меню **File**.
2. Выберите пункт меню **Host Network Manager...**.
3. Нажмите на кнопку **Create**.
4. Подтвердите создание виртуального сетевого интерфейса.
   Нажмите на кнопку **Yes**.
5. Создайте еще 2 виртуальных сетевых интерфейса.
   Для этого повторите пункты 3 и 4.
6. Нажмите на кнопку **Properties**.
7. Выберите виртуальный сетевой интерфейс
   **VirtualBox Host-Only Ethernet Adapter**.
8. Задайте для виртуального сетевого интерфейса IP-адрес **192.168.3.6**.
9. Задайте для виртуального сетевого интерфейса маску подсети **255.255.255.0**.
10. Снимите галочку **Enable** для **DHCP Server**.
11. Нажмите на кнопку **Apply**.
12. Подтвердите изменение параметров виртуального сетевого интерфейса.
    Нажмите на кнопку **Yes**.
13. Аналогичным образом измените оставшиеся виртуальные сетевые интерфейсы:
    - **VirtualBox Host-Only Ethernet Adapter #2** ->
      **192.168.8.6** / **255.255.255.0**
    - **VirtualBox Host-Only Ethernet Adapter #3** ->
      **192.168.9.6** / **255.255.255.0**
    - **VirtualBox Host-Only Ethernet Adapter #4** ->
      **192.168.10.6** / **255.255.255.0**
14. Нажмите на кнопку **Close**.

### 1.3 Импорт виртуальной машины VirtualBox

![Раздел 1.3 Рисунок 1][Section_1.3_Figure_1]

![Раздел 1.3 Рисунок 2][Section_1.3_Figure_2]

![Раздел 1.3 Рисунок 3][Section_1.3_Figure_3]

![Раздел 1.3 Рисунок 4][Section_1.3_Figure_4]

Для импорта виртуальной машины **VirtualBox** выполните следующие действия:

1. Выберите меню **File**.
2. Выберите пункт меню **Import Appliance...**.
3. В меню **Source** выберите локальную файловую систему **Local File System**.
4. В меню **File** укажите путь к файлу в хостовой ОС.
   Данный файл должен содержать импортируемый образ.
5. Нажмите на кнопку **Next**.
6. В меню **Machine Base Folder** укажите путь к директории куда будет
   импортирована виртуальная машина. При этом в указанной директории будет
   создана поддиректория с именем импортируемой виртуальной машины.
7. В меню **MAC Address Policy** выберите
   **Generate new MAC addresses for all network adapters**.
8. В меню **Additional Options** поставьте галочку
   **Import hard drives as VDI**.
9. Нажмите на кнопку **Import**.
10. После импортирования при необходимости измените
    параметры виртуальной машины. См. подпункт 2.2.

### 1.5 Установка VS Code Server

Для установки **VS Code Server** выполните следующие действия:

1. Скачайте с репозитория на **Yandex.Disk** архив **vscode-server.tar.gz**.
2. Поместите скачанный архив в общую папку. См. подпункт 2.2.4.
3. Запустите виртуальную машину **VirtualBox**. См. подпункт 1.4.
4. После запуска виртуальной машины **VirtualBox** авторизуйтесь.
   В приглашении **angtel login:** введите **root**.

   ```console
   AngtelOS 3.6.0 angtel

   angtel login: root
   Last login: Wed Feb 19 08:10:06 UTC 2020 from 192.168.3.6 on pts/0
   root@angtel:~#
   ```

5. Распакуйте архив **vscode-server.tar.gz** с помощью следующей команды:

   ```console
   tar -zxf ~/share/vscode-server.tar.gz -C ~/
   ```

### 1.7 Подключение VS Code Client к VS Code Server

![Раздел 1.7 Рисунок 1][Section_1.7_Figure_1]

![Раздел 1.7 Рисунок 2][Section_1.7_Figure_2]

![Раздел 1.7 Рисунок 3][Section_1.7_Figure_3]

![Раздел 1.7 Рисунок 4][Section_1.7_Figure_4]

![Раздел 1.7 Рисунок 5][Section_1.7_Figure_5]

![Раздел 1.7 Рисунок 6][Section_1.7_Figure_6]

![Раздел 1.7 Рисунок 7][Section_1.7_Figure_7]

![Раздел 1.7 Рисунок 8][Section_1.7_Figure_8]

Для подключения **VS Code Client** к **VS Code Server**
выполните следующие действия:

1. Перейдите в директорию с распакованным архивом для **VS Code Client**.
2. Запустите приложение **VS Code**.
3. В левом нижнем углу нажмите на кнопку **Open a remote window**.
4. Во всплывающем вверху меню выберите **Remote-SSH: Connect to Host...**.
5. Введите имя пользователя и адрес удаленного хоста **root@192.168.3.8**.
6. Если подключение осуществляется первый раз, то во всплывающем меню
   необходимо выбрать **Continue**.
7. При необходимости разрешить приложению **VS Code** доступ к
   **Private networks** в **Firewall**.
8. В случае неуспешного соединения нажмите на кнопку **Retry**.  
   Также можно нажать на кнопку **Close Remote** и повторить пункты 3-7.  
   Если причиной неуспешного подключения является изменившийся
   в виртуальной машине SSH-ключ, то необходимо на хостовой машине
   удалить файл **C:\\Users\\mtuser\\.ssh\\known_hosts**.
9. Индикатором успешного соединения служит кнопка в левом нижнем углу.  
   Она должна отображать надпись **SSH: 192.168.3.8**.

Запустите средство виртуализации **VirtualBox**.

1. Нажмите на кнопку **New**.

3. Выберите расположение виртуальной машины в хостовой ОС.
4. Выберите тип гостевой ОС **Linux**.

#### 3.11.1 Подключение к гостевой ОС через последовательный порт

Перед выполнением подключения необходимо убедиться в наличии
утилиты **minicom**. Для этого требуется выполнить команду:

```console
which minicom
```

Если предыдущая команда не отображает путь к исполняемому файлу,
то необходимо выполнить установку утилиты **minicom**:

```console
sudo apt-get update
sudo apt-get -y install minicom
sudo usermod -a -G dialout $USER
```

Подключение к гостевой ОС через последовательный порт
выполняется с помощью следующей команды:

```console
minicom -D unix\#/tmp/com1 -b 115200
```

После подключения необходимо авторизоваться в гостевой ОС.

Завершение текущего соединения осуществляется путем
нажатия комбинации клавиш **Ctrl+A Q**. Далее выбирается пункт **Yes**.

#### 3.11.2 Подключение к гостевой ОС через SSH

Подключение к гостевой ОС через SSH выполняется с помощью следующей команды:

```console
ssh root@192.168.3.8
```

В случае первого подключения к гостевой ОС необходимо
подтвердить установку соединения напечатав **yes**.

```console
The authenticity of host '192.168.3.8 (192.168.3.8)' can't be established.
ECDSA key fingerprint is SHA256:d/E0on+tTs94Fbqf3ZHfWCCf8hiCF1gg9PJf70Uk30s.
Are you sure you want to continue connecting (yes/no)? yes
Warning: Permanently added '192.168.3.8' (ECDSA) to the list of known hosts.
```

Завершение текущего соединения осуществляется путем
ввода в гостевой ОС команды **exit**.

#### 3.11.3 Подключение к гостевой ОС через Telnet

Подключение к гостевой ОС через Telnet выполняется с помощью следующей команды:

```console
telnet 192.168.3.8
```

После подключения необходимо авторизоваться в гостевой ОС.

Завершение текущего соединения осуществляется путем
ввода в гостевой ОС команды **exit**.
